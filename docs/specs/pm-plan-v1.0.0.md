## 2. MVPスコープ定義

### MVP必須機能（Phase 1 -- 7 Sprint / 2026年4月中旬〜7月末）

| # | 機能名 | 除外した場合のビジネスリスク |
|---|--------|--------------------------|
| 1 | Better Auth認証（メール/パスワード） | ユーザーデータの永続化不可。無料枠の制御不能 |
| 2 | 固定シナリオ5本 + 生活ノイズランダム変動 | コアバリュー（臨床判断シミュレーション）が提供不可 |
| 3 | 役割選択（AT / S&C） | コア学習ループのStep 1が成立しない |
| 4 | エクササイズDB（300-500件 + pgvector RAG） | メニュー処方の選択肢がなく、シミュレーションの実用性が消失 |
| 5 | メニュー処方UI（検索・フィルタ） | ユーザーが処方行為を行えず、学習ループが回らない |
| 6 | LLMエージェント4体（セリフ生成のみ） | 対立意見・クライアントプレッシャーの欠如でジレンマが発生しない。「フライトシミュレータ」としての臨場感が消失 |
| 7 | テキストディベートUI | AIエージェントとの議論ができず学習体験が成立しない |
| 8 | DILEMMAラッパーAPI（生活ノイズ→Z-score変換 + 状態遷移管理） | V7推論エンジンとの接続不可。シナリオの時間進行が実現不可 |
| 9 | V7推論エンジン連携（結果の確定的計算） | 科学的根拠に基づく結果が出せず、教育ツールとしての信頼性が崩壊 |
| 10 | レーダーチャート（Recharts） | 結果の可視化ができず、ユーザーの理解度が大幅低下 |
| 11 | AIメンターAAR + cause_breakdown | 「なぜそうなったか」の説明がなく、学習効果が半減 |
| 12 | タイムトラベル（スライダー + What-If再実行） | Kolb学習ループの完結不可。「修正して再挑戦」ができない |
| 13 | 比較レーダーチャート（オーバーレイ） | What-Ifの効果が定量的に把握できず、タイムトラベル機能の価値が半減 |
| 14 | セッションログ保存・閲覧 | 学習記録が消失。ダッシュボードのスキルレーダーも機能しない |
| 15 | Freeプラン（月3シナリオ制限） | ユーザー獲得導線が消失。体験なしでの有料転換は期待できない |
| 16 | Stripe課金（クレジット購入） | 収益化手段がなく、LLMコストを回収できない |
| 17 | ダッシュボード（進捗・履歴・スキルレーダー） | 学習の継続動機が弱く、リテンション低下 |

### Phase 2以降（理由付き）

| 機能名 | Phase 1では除外した理由 |
|--------|----------------------|
| 3Dアバター / 代償動作アニメーション | Python CVエンジン連携が必要。テキスト+レーダーチャートでMVP価値は検証可能 |
| 教育機関向けクラス管理 | B2B機能。まず個人ユーザーでの価値検証が先 |
| カスタムシナリオ生成 | 固定5シナリオで学習価値を検証してからの拡張 |
| 英語対応 | 日本市場（PT養成スクール・JATO基準）が初期ターゲット |
| SSO認証 | 教育機関向けB2B対応時に追加 |

### スコープ外（理由付き）

| 機能名 | 除外理由 |
|--------|---------|
| マルチプレイヤー対戦 | インフラ・UX設計の複雑性が大。Phase 3以降の差別化機能 |
| データフライホイール | 十分な意思決定ログ蓄積後に実装すべき。Phase 3 |
| B2B Enterprise / API公開 | 個人ユーザーでPMF検証後に着手 |
| LMS連携 | B2B機能。教育機関との契約後に開発 |

## 3. KPIツリー

### 最上位目標: PoCで教育効果を実証し、9月ローンチ時に10校導入 + 月間ARR 30万円

```
├── 獲得KPI
│   ├── PoC参加者数: 50名（8月） [計測: user_profiles COUNT]
│   ├── ローンチ月の新規登録数: 200名（9月） [計測: users COUNT WHERE created_at in 9月]
│   └── スクール導入数: 10校（9月末） [計測: user_profiles WHERE affiliation IS NOT NULL, DISTINCT]
│
├── 活性化KPI
│   ├── シナリオ完走率: 70%以上 [計測: scenario_sessions WHERE status='completed' / 全sessions]
│   ├── Day 7 継続率: 40%以上 [計測: 登録7日後にセッション実行したユーザー割合]
│   ├── 平均セッション数/ユーザー/月: 5回以上 [計測: scenario_sessions GROUP BY user_id, month]
│   └── What-If利用率: 50%以上 [計測: whatif_comparisons存在するsession / 全session]
│
├── 学習効果KPI（教育ツール固有）
│   ├── AAR閲覧完了率: 80%以上 [計測: aar_records閲覧ログ]
│   ├── 2回目以降の改善率: 60% [計測: user_progress.avg_risk_score の経時変化]
│   └── PoC満足度スコア: 4.0/5.0以上 [計測: PoC後アンケート]
│
└── 収益KPI
    ├── Free→クレジット購入転換率: 15%以上 [計測: subscriptions WHERE plan!='free' / 全users]
    ├── 月間クレジット購入額: 30万円（ローンチ3ヶ月後） [計測: Stripe revenue]
    ├── 1ユーザーあたり月間消費額: 1,500円 [計測: usage_tracking.llm_tokens_used → 金額換算]
    └── 月次解約率（Churn Rate）: 10%以下 [計測: 前月アクティブで今月非アクティブ / 前月アクティブ]
```

## 5. リスクと対策

| リスク | 影響度 | 対策 |
|--------|-------|------|
| V7推論エンジンのDILEMMA向けチューニング不足 | 高 | Sprint 3で早期統合テスト。生活ノイズ→Z-score変換の精度をPoC前に検証 |
| Gemini Free Tier のレートリミット（15req/min） | 中 | ユーザー別キュー制御。同時利用50人超で有料Tier移行計画を用意 |
| エクササイズDB 300-500件の構築工数 | 中 | 既存Pace-platform DBからPT向けを抽出。Phase 1は200件からスタート、Sprint 2-3で段階的に拡充も許容 |
| Neon Free枠（0.5GB）の容量不足 | 低 | PoC時点でのデータ量を計測。100ユーザー超でNeon Pro移行 |
| PoC参加校の確保 | 高 | 5月からPT養成スクールへのアプローチ開始（ケイ/ミオ担当） |
